# Story 1.1: Monorepo workspace scaffold

Status: review

## Story

As an engineer,
I want a monorepo workspace matching architecture §18,
so that every subsequent story lands in the correct location.

## Acceptance Criteria

1. Root workspace (`apps/web`, `apps/api`, `packages/domain-contracts`, `packages/scoring-core`, `infra/`, `docs/`) exists with a shared package-manager workspace manifest.
2. `tsconfig.base.json` enforces TypeScript strict mode and is extended by every workspace.
3. Internal packages are resolvable via workspace protocol in `apps/web` and `apps/api`.
4. Repo-root `README.md` documents workspace layout and dev/build commands.

### Completion Notes List

- **AC1 — workspace manifest:** `pnpm-workspace.yaml` declares `apps/*` and `packages/*`. Root `package.json` pins `packageManager: "pnpm@9.15.0"` and exposes repo-wide scripts (`dev`, `build`, `typecheck`, `lint`, `test`, `test:scaffold`). `engines` field enforces Node ≥20.11.0 / pnpm ≥9. `.npmrc` pins `engine-strict`, `save-exact`, `link-workspace-packages`, `prefer-workspace-packages` for deterministic, strict installs. `.gitignore` covers `node_modules`, build output, env files, and pnpm-store.
- **AC2 — strict TS:** `tsconfig.base.json` sets `strict: true` plus `noUncheckedIndexedAccess`, `noImplicitOverride`, `exactOptionalPropertyTypes`, `noFallthroughCasesInSwitch`, `forceConsistentCasingInFileNames`, `isolatedModules`. Every workspace `tsconfig.json` extends `../../tsconfig.base.json` verbatim — asserted by test.
- **AC3 — workspace protocol:** `apps/web` declares `@fcm/domain-contracts: "workspace:*"`. `apps/api` declares both `@fcm/domain-contracts` and `@fcm/scoring-core` via `workspace:*`. Placeholder `src/index.ts` in each app imports from both internal packages and typechecks clean. `pnpm install` produced live workspace links verified via `fs.realpathSync` in the scaffold tests.
- **AC4 — README:** documents repo shape (ASCII tree), flat workspace-paths list (prose, contiguous — consumed by automated check), internal-package table, prerequisites (Node / pnpm), install command, and a full dev / build / typecheck / test / scaffold-test command table, plus workspace-scoped examples using `pnpm --filter`. Closes with pointers to architecture §18 / §5.1 / §5.4 and to the follow-on scaffold stories (1-2, 1-3, 1-4).
- `apps/web` and `apps/api` are intentionally minimal placeholders; full framework init happens in Stories 1-3 (Next.js) and 1-2 (NestJS). Their current `src/index.ts` files exist only to prove workspace-protocol resolution and strict-mode compilation.
- Scaffold guardrail: `pnpm run test:scaffold` runs 12 assertions covering all 4 ACs using Node's built-in `node:test` (zero added deps). Safe to keep as a CI invariant for subsequent stories.
